# Office Agent 需求文档

> 愿景：一个能理解自然语言、自主使用工具、处理各种常见办公数据需求的智能 Agent。

---

## 1. 用户画像

- **非技术型办公人员**：会用 Excel/Word/PPT，不会写公式或脚本，日常处理报表、合同、汇报材料
- **数据处理密集型角色**：财务、HR、行政、项目经理，每天需要在多个文档间搬运和加工数据
- **核心诉求**：用一句话描述需求，上传文件，拿到结果。不想学习任何工具或配置

---

## 2. 核心能力需求

### 2.1 文档理解

| 需求 | 用户典型表述 | 说明 |
|------|-------------|------|
| 内容摘要 | "帮我总结这份报告的要点" | 读取文档，输出结构化摘要 |
| 内容问答 | "这份合同的违约金是多少？" | 理解文档内容，定位并回答具体问题 |
| 结构提取 | "列出这个文档里所有的表格" | 识别文档中的段落、表格、图片、标题层级等结构 |

### 2.2 数据提取与整理

| 需求 | 用户典型表述 | 说明 |
|------|-------------|------|
| 表格提取 | "把这个 PDF 里的表格导出成 Excel" | 从任意格式中识别表格并结构化输出 |
| 字段提取 | "从这些简历里提取姓名、电话、学历" | 批量文档 → 结构化数据 |
| 信息汇总 | "把这 10 个分公司的报表合并成一张总表" | 多文件同结构数据合并 |
| 数据清洗 | "这列日期格式不统一，帮我标准化" | 识别数据问题并修复 |

### 2.3 模板填充与文档生成

| 需求 | 用户典型表述 | 说明 |
|------|-------------|------|
| 合同/通知生成 | "用这个模板，根据 Excel 里的人员信息生成 50 份通知" | 模板 + 数据源 → 批量文档 |
| 格式转换 | "把这个 Markdown 转成带格式的 Word" | 跨格式内容迁移 |
| 报告生成 | "根据这些数据写一份季度分析报告" | 数据 → 带图表和文字的完整报告 |

### 2.4 数据分析与计算

| 需求 | 用户典型表述 | 说明 |
|------|-------------|------|
| 统计分析 | "这份销售数据的月均增长率是多少？" | 对表格数据做计算和统计 |
| 对比分析 | "对比这两个月的报表，哪些指标变化最大？" | 跨文件/跨时段的数据对比 |
| 异常检测 | "帮我看看这列数据有没有异常值" | 数据质量检查 |

### 2.5 文档对比与校验

| 需求 | 用户典型表述 | 说明 |
|------|-------------|------|
| 差异对比 | "这两版合同有什么区别？" | 逐段/逐句对比，标注差异 |
| 数据校验 | "检查一下填好的表有没有漏填的字段" | 模板字段完整性检查 |
| 一致性检查 | "汇总表的数字和各分表加起来对得上吗？" | 跨文档数据一致性验证 |

### 2.6 演示文稿处理

| 需求 | 用户典型表述 | 说明 |
|------|-------------|------|
| PPT 内容提取 | "把这份 PPT 的每页要点列出来" | 读取幻灯片内容和结构 |
| PPT 生成 | "根据这份报告做一个 10 页的汇报 PPT" | 从文档内容生成演示文稿 |
| PPT 内容更新 | "把第三页的数据换成最新版 Excel 里的数字" | 定向修改幻灯片内容 |

## 3. Agent 行为需求

### 3.1 自主推理与规划

- Agent 必须能根据用户意图**自主决定**执行步骤，而不是依赖预设的流程模板
- 对于复杂需求，Agent 应该能分解为多个子任务并逐步执行
- 执行过程中遇到问题时，Agent 应该能**自主调整策略**（而非死板重试）

### 3.2 工具使用

- Agent 拥有一组工具（读取、提取、填充、写入、分析等），由 LLM 自主决定调用哪个工具、传什么参数
- 工具列表是可扩展的，新增工具不应需要修改 Agent 核心逻辑
- 工具调用失败时，Agent 应该理解错误原因并尝试替代方案

### 3.3 代码执行

- Agent 能够生成并执行 Python 代码，用于数据分析、计算、格式转换等场景
- 典型场景：统计分析（均值/增长率/占比）、数据清洗（去重/格式化/类型转换）、复杂计算（透视表/合并/条件筛选）
- 代码在隔离的沙箱环境中执行，有超时和资源限制
- 执行结果（返回值、stdout、报错）作为 observation 回传给 Agent，Agent 根据结果决定下一步
- 代码执行不是独立工具，而是 Agent 工具集中的一种——由 Agent 自主判断何时需要写代码

### 3.4 多轮对话

- 用户可以在上一轮结果基础上追问、修改、补充，无需重复上下文
- Agent 维护完整的对话历史（用户消息 + Agent 回复 + 工具调用记录），跨轮次可用
- 支持多轮中追加新文件或替换已有文件
- 典型场景：
  - 第一轮："把这个 Excel 的数据提取出来" → 第二轮："再帮我做个图表" → 第三轮："图表放到 PPT 里"
  - 第一轮："用模板生成合同" → 第二轮："把甲方名称改成 XX 公司" → 第三轮："再生成 10 份"
- Agent 应能区分"新任务"和"在已有上下文上继续"，合理决定是复用还是重置状态

### 3.5 上下文记忆

- Agent 在单次任务执行过程中维护完整的推理历史（thought + action + observation）
- Agent 应该能引用之前步骤的中间结果（比如"用刚才提取的数据去填充模板"）
- 对话历史在会话期间持久化，会话结束后可选择性保存

### 3.6 透明性

- Agent 的每一步推理和工具调用对用户可见
- 最终结果附带执行过程摘要：做了什么、为什么这么做、耗时多少
- 失败时说明原因和已尝试的替代方案

## 4. 交互需求

### 4.1 输入方式

- 自然语言描述需求（中文/英文）
- 上传一个或多个文件（Word/Excel/PDF/PPT/CSV/文本）
- 支持多轮对话中追加文件或修改需求

### 4.2 输出方式

- 结构化文本结果（摘要、回答、分析结论）
- 生成的文件（填充后的文档、导出的表格、生成的报告）
- 执行过程日志（推理链、工具调用记录）

### 4.3 响应模式

- 长任务实时推送进度（当前在做什么、已完成几步）
- 支持中途取消
- 最终结果包含成功/失败状态及原因

## 5. 开发者调试功能

### 5.1 LLM 通信透明

开发者需要能直观地看到 Agent 与 LLM 之间的每一次通信，而非只有代码层面的黑盒。具体包括：

**查看发送给 LLM 的完整 messages：**
- system prompt 的完整内容
- 每轮 user/assistant 消息
- 工具调用请求（function call）及其参数
- 消息的 token 用量估算

**查看 LLM 返回的原始 response：**
- 完整的 response 文本（包括 thought / reasoning）
- 工具调用指令（tool_calls）及其参数
- finish_reason、usage（prompt_tokens / completion_tokens）
- 响应耗时

### 5.2 调试面板

前端提供一个开发者面板（可通过开关启用/关闭），用于实时观察 Agent 的内部运作：

- **消息流视图**：以时间线形式展示每一轮 LLM 交互，每条消息可展开查看完整 JSON
- **工具调用链**：展示每次工具调用的输入参数和返回结果
- **状态快照**：在每个关键节点（规划完成、每步执行后、验证后）展示 Agent 的内部状态
- **性能指标**：每轮 LLM 调用的 token 消耗和响应时间，整个任务的总耗时和总 token 消耗

### 5.3 设计原则

- 调试信息对终端用户默认隐藏，通过显式开关开启
- 开发者面板的数据与正常业务流完全解耦，不影响 Agent 执行性能
- 所有调试数据在前端渲染，不需要额外的后端查询接口

## 6. 文件格式支持

| 格式 | 读取 | 写入 | 备注 |
|------|:----:|:----:|------|
| .docx / .doc | ✅ | ✅ | Word 文档，支持模板占位符替换 |
| .xlsx / .xls / .csv / .tsv | ✅ | ✅ | 表格数据，支持多 Sheet、公式区域 |
| .pdf | ✅ | ❌ | 文本型 PDF |
| .pptx | ✅ | ✅ | 演示文稿，逐幻灯片操作 |
| .txt / .md | ✅ | ✅ | 纯文本，自动编码检测 |

---

## 7. 质量需求

- **准确率**：字段提取、模板填充的正确率应 ≥ 95%（结构化场景）
- **鲁棒性**：对格式不规范的文档（合并单元格、嵌套表格、乱码编码）有合理的降级处理
- **幂等性**：相同输入多次执行应产生一致的结果
- **可追溯**：每个输出文件可追溯到生成它的输入文件和操作步骤

---

## 8. 不在范围内（v1）

- 实时协作（多人同时编辑）
- 图片/扫描件 OCR（后续可扩展）
- 音视频处理
- 邮件收发
- 日历/会议管理
- 浏览器自动化（爬取网页数据）
